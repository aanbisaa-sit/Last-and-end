# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Last and End, please report it by emailing aanbisaa@example.com.

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and work with you to address the issue promptly.

## Security Best Practices

When using Last and End:
- Keep timeout values reasonable to prevent resource exhaustion
- Validate endpoint URLs from untrusted sources
- Use HTTPS endpoints when possible
- Limit retry counts to prevent excessive network traffic
- Store configuration files securely with appropriate permissions
